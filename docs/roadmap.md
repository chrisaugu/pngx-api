# NUKU-API (formerly PNGX-API) Roadmap

Welcome to the NUKU-API roadmap! This document outlines our plans for the future of the project. It is intended to give contributors and users a clear idea of what we are working on and where the project is heading.

Please note that this roadmap is a living document and is subject to change based on community feedback, shifting priorities, and emerging technologies.

## Current Focus (Short-Term Goals)

* **Stabilize v3 Release:** Ensure all features in the v3 release are stable, fully tested, and documented.
* **Enhance Documentation:** Continue improving the API documentation, including adding more examples and use cases.
* **Community Building:** Encourage community contributions, triage issues promptly, and establish a clear governance model.
* **Implement Rate Limiting and Security Enhancements:** Refine rate limiting strategies and ensure all endpoints are secure against common vulnerabilities.

## Mid-Term Goals (Next 6-12 Months)

* **Webhooks Integration:** fully finalize and document the webhooks system to allow external services to be notified of market events in real-time.
* **Extended Historical Data:** Partner with providers or build tools to backfill more historical stock data and ensure its accuracy.
* **GraphQL API Formulation:** Explore offering a GraphQL endpoint to allow clients to specify exactly what data they need, reducing over-fetching.
* **SDK Development:** Create official SDKs for popular languages (e.g., Python, Ruby, Go) to make it easier for developers to integrate NUKU-API into their applications.

## Long-Term Vision (1+ Years)

* **Advanced Analytics and Insights:** Provide endpoints that offer computed analytics, such as moving averages, volatility metrics, and technical indicators.
* **Machine Learning Support:** Explore datasets and formats optimized for machine learning models to predict market trends or detect anomalies.
* **Global Market Expansion:** While currently focused on PNGX, explore the architecture needed to support other regional exchanges if there is demand.
* **Decentralized Data Storage:** Investigate the feasibility of using decentralized storage solutions for archiving historical data to ensure immutability and high availability.

## How to Contribute

We value community input! If you have ideas for the roadmap or want to help implement any of these features:

1. **Check the Issue Tracker:** Look for issues tagged with `roadmap` or `enhancement`.
2. **Open a Discussion:** Propose new ideas in the GitHub Discussions to get feedback before writing code.
3. **Submit a Pull Request:** Follow the guidelines in our [CONTRIBUTING.md](../CONTRIBUTING.md) to submit your changes.

Thank you for your interest in improving NUKU-API!
